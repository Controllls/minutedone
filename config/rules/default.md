# 기본 추출 규칙

이 파일은 회의록에서 업무를 추출할 때 LLM에게 전달되는 규칙입니다.
회사/팀에 맞게 수정하세요.

---

## 담당자 규칙

- 담당자가 명시되지 않고 전체 팀에 해당하는 업무는 assignee를 "전체"로 설정
- 호칭(팀장, 매니저 등)이 붙어도 이름만 추출 (예: "박팀장" → "박팀장" 유지)

## 날짜 규칙

- 별도 언급이 없는 배포/릴리즈 관련 업무는 다음 주 금요일을 기본 마감으로 설정
- "이번 스프린트"는 이번 주 금요일 마감으로 처리
- "다음 스프린트"는 다음 주 금요일 마감으로 처리

## 업무 분류 규칙

- 디자인, 개발, QA, 배포가 함께 언급되면 하나의 parent 아래 sub로 묶을 것
- "기획", "설계", "리뷰"가 포함된 업무는 parent로 분류
- 단순 보고/공유 업무 (회의록 정리, 공지 등)는 독립 sub로 처리

## 용어 정의

- STT: 음성 녹음을 텍스트로 변환하는 작업
- RAG: AI가 저장된 문서를 검색해서 답변하는 기능
