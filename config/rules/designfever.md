# DesignFever 추출 규칙

DesignFever 팀 회의록 전용 규칙입니다. default.md 규칙에 추가로 적용됩니다.

---

## 담당자 규칙

- 디자이너 이름이 언급되면 assignee에 그대로 기입
- "DF", "디자인피버" 등 회사명만 언급된 경우 assignee를 "전체"로 설정
- PL(프로젝트 리드), CD(크리에이티브 디렉터), PM 직책이 붙어도 이름만 추출

## 날짜 규칙

- 클라이언트 제출/발표 관련 업무는 언급된 날짜를 엄격히 따를 것
- "시안 마감", "시안 제출"은 별도 날짜 언급이 없으면 다음 주 수요일로 설정
- "최종 납품", "최종 전달"은 별도 날짜 언급이 없으면 다음 주 금요일로 설정
- "오늘 중", "금일"은 오늘 날짜로 처리

## 업무 분류 규칙

- 리서치 → 기획 → 시안 → 수정 → 최종 납품 흐름은 하나의 parent 아래 sub로 묶을 것
- 브랜딩, UI, 영상, 인쇄물 등 산출물 유형이 다르면 각각 별도 parent로 분리
- 클라이언트 미팅/발표 준비는 독립 parent로 처리
- 내부 피드백·검토 업무는 sub로 처리

## 용어 정의

- 시안: 디자인 초안 (draft)
- 목업: 실제 환경에 적용한 시뮬레이션 이미지
- 가이드라인: 브랜드/디자인 규칙 문서
- PT: 클라이언트 대상 프레젠테이션
- BI: 브랜드 아이덴티티
- GUI: 그래픽 유저 인터페이스 디자인
